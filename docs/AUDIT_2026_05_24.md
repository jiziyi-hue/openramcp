## 1. Daemon 自动操作链 (完整 trace)

### 1.1 主循环
- `_loop()` (tactical.py:2104): 后台线程 0.6s 一 tick (`POLL_INTERVAL_S`).
- `_tick()` (tactical.py:2112): 拉 world 快照 → 跑 `_resolve_dynamic_forces` → 各 mission `_run_*` → support pairing → pending retry → emit events.

### 1.2 Mission 注册 + 首次分发
- `register_assault()` (tactical.py:727): 建 `Assault` 对象, `dispatched_initial=False`, 加入 `_assaults` list, mission_id 返回 LLM.
- `_run_assault()` (tactical.py:2414): 每 tick 跑:
  1. **首次** (2442): `if not a.dispatched_initial` → 发 `set_stance(AttackAnything)` + `move(attack_move=True)` to final_target → 置 `dispatched_initial=True`
  2. 后续 tick: retreat check / cohesion gate / engage picker / kite logic

### 1.3 Atomic 命令向 transport
Daemon 通过 `transport.send_command()` 发:
- `set_stance` — AttackAnything / Defend / HoldFire / ReturnFire
- `move` — `attack_move: true` 时为冲锋, false 普通行军
- `attack` — `target_id`
- `stop` — cohesion halt / 撤退中
- `get_state` — 查询

**Interpreter** (`interpreter.py`) 仅注册 mission, **不发** atomic — `feedback_llm_only_states_not_commands.md` 标的 root bug 已修 ✓.

### 1.4 Cohesion Gate (tactical.py:2484)
- 算 active 部队 centroid + 沿目标方向 forward vec
- 标 **vanguard outliers** (spread > 9 cells AND along > 0)
- 发 `stop` halt 之, 后队继续 attack_move
- 假设: 后队"在 attack_move 中追上". **若后队从未收到 attack_move (见 §2.1), 假设破裂**.

### 1.5 Engage Picker (tactical.py:2833+)
- ENGAGE_RADIUS=8 / EXTENDED_ENGAGE_RADIUS=18 (后者仅 priority ≥ 80 单位如 v2rl/arty/tsla 触发)
- `_other_locked_targets()` 给被其他 mission 锁的 target 加 0.5x 惩罚 — 多 mission 协调 ✓

### 1.6 Wave Promotion (tactical.py:2368)
- `wave_size` 设 → 先派 wave_size 个, 余下待机
- 后续波触发:
  - `wave_interval_s` 时间到
  - 第一波 bleeding (avg_hp < 0.65 AND alive < wave_size × 0.5) → 提前
- 选拣顺序: unit_strength 排序

### 1.7 Retreat (tactical.py:2762)
- 触发: `hp < RETREAT_HP_THRESHOLD` (0.3 默认, 各 alert state 不同)
- 发 `set_stance(HoldFire)` + move 回 base
- 恢复: `hp >= REENGAGE_HP_THRESHOLD` (0.7) + 冷却到期

### 1.8 Defense Perimeter (tactical.py:2930)
- `_run_defense()` 每 tick 扫每 zone (8s cooldown)
- 敌方 mobile 进 radius → 征召半径 1.5x 内非 busy garrison → `attack(target_id)`
- Garrison **不包括** assault/harass/etc force_ids (避免抢兵)

### 1.9 Support Pairing
- `_support.tick()` 1s eval, 不在 0.6s 主 tick
- medic → 半径 10 内 hp<0.7 友军步兵
- mechanic → 同上对车

### 1.10 Pending Queue
- `_tick_pending()` 每 3s 重试 force 解析
- 解析 ≥1 → 调 `_dispatch_auto_mission()` 启动 → 从 pending 移除
- 无 backoff (P2 改进)

## 2. 致命 BUG (实测暴露, 静态扫漏)

### 2.1 ⚠️ **新单位罚站 BUG (P0)**

**代码位置**: `tactical.py:2442-2454`

```python
if not a.dispatched_initial:
    id_list = list(alive_ids)
    self.transport.send_command({"type": "set_stance", ...})
    self.transport.send_command({"type": "move", ...})
    a.dispatched_initial = True
```

**问题**:
- `dispatched_initial` 是 **mission 级** flag, 只发**一次** attack_move
- `_refresh_mission_force()` 每 3s 加新单位到 `mission.force_ids`, force_alive 涨, **但新加 actor 永远没收到 attack_move**
- 新 actor 站家不动, 假设"在 attack_move 中" 实际为假
- Cohesion gate 把他当 rear-straggler 不 halt, 但他根本没在动

**实测**:
- 会话末 mission 7 force_alive=95, 但 30+ e3 + 部分 4tnk 站家门口 (58-61, 20-24)
- cancel + 重派 mission 8 → 全 force 重新解析 + 重发 attack_move → 16s 内推平敌方

**修复方案**:
```python
# 改成 per-actor tracking
if not hasattr(a, 'dispatched_actors'):
    a.dispatched_actors = set()
new_actors = alive_ids - a.dispatched_actors
if new_actors:
    new_list = list(new_actors)
    self.transport.send_command({"type": "set_stance", "unit_ids": new_list, ...})
    self.transport.send_command({"type": "move", "unit_ids": new_list, ...})
    a.dispatched_actors.update(new_actors)
```

### 2.2 ⚠️ **同 Pool 多 Mission 抢单 BUG (P0)**

**代码位置**: `interpreter.py` pincer 实现 + `tactical.py:_refresh_mission_force`

**问题**:
- pincer 派 mission 5 (left filter combat_mobile) + mission 6 (right filter combat_mobile)
- 两 mission `force_spec` 解析同 pool, **每 tick 单位被两边重新争夺**
- 视觉表现: 部队反复 left → right → left 重指, "乱晃"
- cohesion_halts 暴涨 (303 → 773 in 60s)

**修复方案**:
- mission 注册时 claim 单位, 其他 mission 用 `exclude_ids` 真排除
- 或 pincer 改为单 mission + 内部 split (分两路 force_ids 锁定)

### 2.3 ⚠️ Cohesion Gate 假设破裂 (P1)

**代码位置**: `tactical.py:2480-2516`

- 假设后队在 attack_move 追赶
- 实际 §2.1 bug 让新单位无 attack_move
- 加上 cohesion 误判 = 永久 halt

**修复**: 修 §2.1 同时根治此问题.

### 2.4 After-Action units_killed_estimate 总返 0 (P1)

- 实测 mission 4/5/6/7 战报 `units_killed_estimate=0` 全部
- `_estimate_kills()` (tactical.py:1780) 实现可能漏 enemy units_lost 跟踪
